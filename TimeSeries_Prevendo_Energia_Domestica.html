Métrica de Avaliação¶

Uma previso ser composta por sete valores, um para cada dia da semana seguinte.

comum em problemas de previso em sries temporais avaliar cada etapa do tempo previsto separadamente. Isso til por alguns motivos:

  • Para avaliar um lead-time especfico (por exemplo, um dia versus trs dias).
  • Para avaliar modelos em diferentes lead-time (por exemplo, modelos bons em +1 dia versus modelos bons em dias +5).

As unidades da potncia total so quilowatts e seria til ter uma mtrica de erro que tambm estivesse nas mesmas unidades. O erro quadrtico mdio raiz (RMSE) e o erro absoluto mdio (MAE) se enquadram nesta lei, embora o RMSE seja mais comumente usado e ser adotado neste tutorial. Ao contrrio do MAE, o RMSE mais punitivo dos erros de previso.

A mtrica de desempenho para esse problema ser o RMSE para cada lead-time do dia 1 ao dia 7.

Pode ser til resumir o desempenho de um modelo usando uma pontuao nica para auxiliar na seleo do modelo.

Uma pontuao possvel que poderia ser usada seria o RMSE em todos os dias previstos.

A funo assessment_forecasts() abaixo implementar esse comportamento e retornar o desempenho de um modelo com base em vrias previses de sete dias.

In [10]:
# avaliar uma ou mais previses semanais em relao aos valores esperados
def evaluate_forecasts(actual, predicted):
    scores = list()
    # calcular uma pontuao RMSE para cada dia
    for i in range(actual.shape[1]):
        # calcular o MSE
        mse = mean_squared_error(actual[:, i], predicted[:, i])
        # calcular o RMSE
        rmse = sqrt(mse)
        # guardar
        scores.append(rmse)
    # calcular todo RMSE
    s = 0
    for row in range(actual.shape[0]):
        for col in range(actual.shape[1]):
            s += (actual[row, col] - predicted[row, col])**2
    score = sqrt(s / (actual.shape[0] * actual.shape[1]))
    return score, scores

A execuo da funo retornar primeiro o RMSE geral, independentemente do dia, depois uma matriz de pontuaes do RMSE para cada dia.

Validação Walk-Forward¶

Os modelos sero avaliados usando um esquema chamado validao direta.

aqui que um modelo necessrio para fazer uma previso de uma semana; em seguida, os dados reais dessa semana so disponibilizados para o modelo, para que possam ser usados como base para fazer uma previso na semana subseqente. Isso mostra como o modelo pode ser usado na prtica, permitindo que eles utilizem os melhores dados disponveis.

Podemos demonstrar isso abaixo com a separao dos dados de entrada e dos dados de sada / previstos.

Entrada, Previso [Semana1] Semana2 [Semana1 + Semana2] Semana3 [Semana1 + Semana2 + Semana3] Semana4 ...

A abordagem de validao passo a passo para avaliar modelos preditivos nesse conjunto de dados fornecida abaixo denominada assessment_model().

Os conjuntos de dados de treino e teste no formato de semana so fornecidos para a funo como argumentos. fornecido um argumento adicional n_input que usado para definir o nmero de observaes anteriores que o modelo usar como entrada para fazer uma previso.

Duas novas funes so chamadas: uma para criar um modelo a partir dos dados de treino chamados build_model() e outra que usa o modelo para fazer previses para cada nova semana, chamada forecast().

Estamos trabalhando com redes neurais e, como tal, geralmente so lentas para treinar, mas rpidas para avaliar. Isso significa que o uso preferido dos modelos constru-los uma vez com dados histricos e us-los para prever cada etapa da validao direta. Os modelos so estticos (ou seja, no so atualizados) durante sua avaliao.

Isso diferente de outros modelos que so mais rpidos de treinar, onde um modelo pode ser reajustado ou atualizado a cada etapa da validao progressiva medida que novos dados so disponibilizados. Com recursos suficientes, possvel usar redes neurais dessa maneira, mas no o usaremos neste tutorial.

def evaluate_model(train, test, n_input):

    model = build_model(train, n_input)
    
    # history  uma lista de dados semanais
    history = [x for x in train]

LSTMs para Multi-Step Forecasting¶

As redes neurais recorrentes, ou RNNs, so projetadas especificamente para trabalhar, aprender e prever dados de sequncia.

Uma rede neural recorrente uma rede neural em que a sada da rede de uma etapa de tempo fornecida como uma entrada na etapa de tempo subsequente. Isso permite que o modelo tome uma deciso sobre o que prever com base na entrada para a etapa de tempo atual e no conhecimento direto do que foi produzido na etapa de tempo anterior.

Talvez o RNN mais bem sucedida e amplamente utilizada seja a rede de memria de longo prazo, ou LSTM, para abreviar. bem-sucedida porque supera os desafios envolvidos no treinamento de uma rede neural recorrente, resultando em modelos estveis. Alm de aproveitar a conexo recorrente das sadas da etapa anterior, os LSTMs tambm possuem uma memria interna que opera como uma varivel local, permitindo que eles acumulem estado na sequncia de entrada.

LSTMs oferecem vrios benefcios quando se trata de previso de sries temporais:

  • Suporte nativo para sequncias: Os LSTMs so um tipo de rede recorrente e, como tal, so projetados para receber dados de sequncia como entrada, ao contrrio de outros modelos em que as observaes de atraso devem ser apresentadas como recursos de entrada.
  • Entradas multivariadas: Os LSTMs suportam diretamente vrias seqncias de entradas paralelas para entradas multivariadas, ao contrrio de outros modelos em que as entradas multivariadas so apresentadas em uma estrutura plana.
  • Sada de vetor: Como outras redes neurais, os LSTMs so capazes de mapear dados de entrada diretamente para um vetor de sada que pode representar vrias etapas do tempo de sada.

Alm disso, foram desenvolvidas arquiteturas especializadas que so projetadas especificamente para fazer previses de sequncia com vrias etapas, geralmente chamadas de previso de sequncia a sequncia, ou seq2seq, resumidamente. Isso til, pois a previso de sries temporais com vrias etapas um tipo de previso seq2seq.

Um exemplo de uma arquitetura de rede neural recorrente projetada para problemas seq2seq o encoder-decoder LSTM.

Um encoder-decoder LSTM um modelo composto por dois submodelos: um chamado codificador que l as seqncias de entrada e o compacta em uma representao interna de comprimento fixo e um modelo de sada chamado decodificador que interpreta a representao interna e a utiliza para prever a sequncia de sada.

A abordagem encoder-decoder para predio de sequncia provou ser muito mais eficaz do que a sada direta de um vetor e a abordagem preferida.

Geralmente, os LSTMs no so muito eficazes em problemas do tipo de auto-regresso. Esses so problemas nos quais a previso da prxima etapa uma funo das etapas recentes.

As redes neurais convolucionais unidimensionais, ou CNNs, provaram ser eficazes no aprendizado automtico de recursos a partir de seqncias de entrada.

Uma abordagem popular tem sido combinar CNNs com LSTMs, onde a CNN um codificador para aprender recursos de sub-sequncias de dados de entrada que so fornecidas como etapas de tempo para um LSTM. Essa arquitetura chamada de CNN-LSTM.

Uma variao de potncia na arquitetura CNN LSTM o ConvLSTM que usa a leitura convolucional das subsequncias de entrada diretamente nas unidades de um LSTM. Essa abordagem se mostrou muito eficaz para a classificao de sries temporais e pode ser adaptada para uso em previses de sries temporais com vrias etapas.

Neste tutorial, exploraremos um conjunto de arquiteturas LSTM para previso de sries temporais em vrias etapas. Especificamente, veremos como desenvolver os seguintes modelos:

  • Encoder-Decoder LSTM para previso em vrias etapas com dados de entrada multivariados.
  • CNN-LSTM Encoder-Decoder para previso em vrias etapas com dados de entrada univariados.

Um modelo LSTM espera que os dados tenham a forma:

[samples, timesteps, features]

Uma amostra ser composta por sete etapas de tempo com um recurso para os sete dias de energia diria consumida.

O conjunto de dados de treinamento possui 159 semanas de dados, portanto, a forma do conjunto de dados de treinamento seria:

[159, 7, 1]

Este um bom comeo. Os dados nesse formato usariam a semana anterior para prever a prxima semana. Um problema que 159 instncias no so muito para treinar uma rede neural.

Uma maneira de criar muito mais dados de treinamento alterar o problema durante o treinamento para prever os prximos sete dias, dados os sete dias anteriores, independentemente da semana.

Isso afeta apenas os dados de treinamento e o problema do teste permanece o mesmo: prever o consumo dirio de energia para a prxima semana padro, dada a semana padro anterior.

Isso exigir uma pequena preparao dos dados de treinamento.

Os dados de treinamento so fornecidos em semanas com oito variveis, especificamente na forma [159, 7, 8]. O primeiro passo achatar os dados para termos oito seqncias de sries temporais.

Precisamos, ento, repetir as etapas do tempo e dividir os dados em janelas sobrepostas; cada iterao se move ao longo de uma etapa e prev os sete dias subsequentes.

Por exemplo:

Entrada, Saida [d01, d02, d03, d04, d05, d06, d07], [d08, d09, d10, d11, d12, d13, d14] [d02, d03, d04, d05, d06, d07, d08], [d09, d10, d11, d12, d13, d14, d15] ...

Podemos fazer isso acompanhando os ndices inicial e final das entradas e sadas medida que iteramos pelo comprimento dos dados achatados em termos de etapas de tempo.

Tambm podemos fazer isso de uma maneira em que o nmero de entradas e sadas seja parametrizado (por exemplo, n_input, n_out) para que voc possa experimentar valores diferentes ou adapt-lo para seu prprio problema.

Abaixo est uma funo chamada to_supervised() que leva uma lista de semanas (histrico) e o nmero de etapas a serem usadas como entradas e sadas e retorna os dados no formato de janela em movimento sobreposto.

def to_supervised(train, n_input, n_out=7):

    data = train.reshape((train.shape[0]*train.shape[1], train.shape[2]))
    X, y = list(), list()
    in_start = 0
    
    # percorrer toda a histria um passo de cada vez
    for _ in range(len(data)):
        # definir o final da sequncia de entrada
        in_end = in_start + n_input
        out_end = in_end + n_out

            x_input = x_input.reshape((len(x_input), 1))
            X.append(x_input)
            y.append(data[in_end:out_end, 0])

        in_start += 1
    return np.array(X), np.array(y)

def evaluate_forecasts(actual, predicted):
    scores = list()
    # calcular uma pontuao RMSE para cada dia
    for i in range(actual.shape[1]):
        # calcular MSE
        mse = mean_squared_error(actual[:, i], predicted[:, i])
        # calcular RMSE
        rmse = sqrt(mse)
        # armazenar
        scores.append(rmse)

    for row in range(actual.shape[0]):
        for col in range(actual.shape[1]):
            s += (actual[row, col] - predicted[row, col])**2
    score = sqrt(s / (actual.shape[0] * actual.shape[1]))
    return score, scores

Encoder-Decoder LSTM com entrada univariada¶

Nesta seo, podemos atualizar o LSTM para usar um modelo de Encoder-Decoder.

Isso significa que o modelo no produzir uma sequncia vetorial diretamente. Em vez disso, o modelo ser composto de dois submodelos, o codificador para ler e codificar a sequncia de entrada e o decodificador que ler a sequncia de entrada codificada e far uma previso em uma etapa para cada elemento na sequncia de sada.

A diferena sutil, pois, na prtica, as duas abordagens de fato prevem uma sada de sequncia.

A diferena importante que um modelo LSTM usado no decodificador, permitindo que ele saiba o que foi previsto para o dia anterior na sequncia e acumule o estado interno enquanto produz a sequncia.

Vamos dar uma olhada em como este modelo definido.

Como antes, definimos uma camada oculta LSTM com 200 unidades. Este o modelo do decodificador que ler a sequncia de entrada e produzir um vetor de 200 elementos (uma sada por unidade) que captura recursos da sequncia de entrada. Usaremos 14 dias de consumo total de energia como entrada.

Usaremos uma arquitetura simples de codificador-decodificador que fcil de implementar no Keras, que tem muita semelhana com a arquitetura de um autoencoder LSTM.

Primeiro, a representao interna da sequncia de entrada repetida vrias vezes, uma vez para cada etapa do tempo na sequncia de sada. Esta sequncia de vetores ser apresentada ao decodificador LSTM.

Em seguida, definimos o decodificador como uma camada oculta LSTM com 200 unidades. importante ressaltar que o decodificador produzir a sequncia inteira, no apenas a sada no final da sequncia, como fizemos com o codificador. Isso significa que cada uma das 200 unidades produzir um valor para cada um dos sete dias, representando a base do que prever para cada dia na sequncia de sada.

Aps esse processo, usaremos uma camada totalmente conectada para interpretar cada etapa do tempo na sequncia de sada antes da camada de sada final. importante ressaltar que a camada de sada prev uma nica etapa na sequncia de sada, nem todos os sete dias de cada vez. Isso significa que usaremos as mesmas camadas aplicadas a cada etapa na sequncia de sada, onde as mesmas camadas de sada totalmente conectadas sero usadas para processar cada etapa do tempo fornecida pelo decodificador. Para conseguir isso, envolveremos a camada de interpretao e a camada de sada em um 'TimeDistributed wrapper' que permite que as camadas envolvidas sejam usadas para cada etapa do decodificador.

Isso permite que o decodificador LSTM descubra o contexto necessrio para cada etapa na sequncia de sada e as camadas densas envolvidas para interpretar cada etapa separadamente, mas reutilizando os mesmos pesos para executar a interpretao. Uma alternativa seria achatar toda a estrutura criada pelo decodificador LSTM e gerar o vetor diretamente. Voc pode tentar isso como uma extenso para ver como ele se compara.

A rede, portanto, gera um vetor tridimensional com a mesma estrutura que a entrada, com as dimenses [amostras, etapas de tempo, recursos].

H um nico recurso, a energia total diria consumida e sempre existem sete recursos. Uma nica previso de uma semana ter, portanto, o tamanho: [1, 7, 1].

Portanto, ao treinar o modelo, devemos reestruturar os dados de sada (y) para ter a estrutura tridimensional em vez da estrutura bidimensional de [amostras, recursos] usada na seo anterior.

Modelo CNN-LSTM Encoder-Decoder com entrada univariada¶

Uma rede neural convolucional, ou CNN, pode ser usada como codificador em uma arquitetura codificador-decodificador.

A CNN no suporta diretamente a entrada de sequncia; em vez disso, uma CNN 1D capaz de ler as entradas da sequncia e aprender automaticamente os recursos mais importantes. Estes podem ser interpretados por um decodificador LSTM como normalmente. Nos referimos a modelos hbridos que usam CNN e LSTM como modelos CNN-LSTM e, neste caso, os estamos usando juntos em uma arquitetura de codificador-decodificador.

A CNN espera que os dados de entrada tenham a mesma estrutura 3D do modelo LSTM, embora vrios recursos sejam lidos como canais diferentes que acabam tendo o mesmo efeito.

Simplificaremos o exemplo e focaremos no CNN-LSTM com entrada univariada, mas ele pode ser facilmente atualizado com o uso de entrada multivariada.

Como antes, usaremos sequncias de entrada compostas por 14 dias de consumo total dirio de energia.

Definiremos uma arquitetura CNN simples, porm eficaz, para o codificador que composto por duas camadas convolucionais, seguidas por uma camada de max pooling, cujos resultados so ento achatados.

A primeira camada convolucional l a sequncia de entrada e projeta os resultados nos mapas de recursos. O segundo executa a mesma operao nos mapas de recursos criados pela primeira camada, tentando amplificar quaisquer recursos salientes. Usaremos 64 features por camada convolucional e leremos as seqncias de entrada com um tamanho de kernel de trs etapas de tempo.

A camada de Max Pooling simplifica as features, mantendo 1/4 dos valores com o maior sinal (mximo). As features mapeadas aps a camada de max pooling so ento achatados em um vetor longo que pode ser usado como entrada no processo de decodificao.

Modelo ConvLSTM Encoder-Decoder com entrada univariada¶

Uma outra extenso da abordagem CNN-LSTM executar as convolues da CNN (por exemplo, como a CNN l os dados da sequncia de entrada) como parte do LSTM para cada etapa do tempo.

Essa combinao chamada de LSTM Convolucional, ou ConvLSTM, para abreviar, e como o CNN-LSTM tambm usada para dados espao-temporais.

Ao contrrio de um LSTM que l os dados diretamente para calcular transies de estado internas, e ao contrrio do CNN-LSTM que est interpretando a sada dos modelos CNN, o ConvLSTM est usando convolues diretamente como parte da leitura de entrada nas prprias unidades LSTM.

A biblioteca Keras fornece a classe ConvLSTM2D que suporta o modelo ConvLSTM para dados 2D. Pode ser configurado para previso de sries temporais multivariadas em 1D.

A classe ConvLSTM2D, por padro, espera que os dados de entrada tenham a forma:

[samples, timesteps, rows, cols, channels]

Onde cada etapa de dados definida como uma imagem de (linhas * colunas).

Estamos trabalhando com uma sequncia unidimensional do consumo total de energia, que podemos interpretar como uma linha com 14 colunas, se assumirmos que estamos usando duas semanas de dados como entrada.

Para o ConvLSTM, isso seria uma leitura nica: ou seja, o LSTM leria uma etapa de 14 dias e executaria uma convoluo entre essas etapas de tempo.

Isto no o ideal.

Em vez disso, podemos dividir os 14 dias em duas subsequncias com durao de sete dias. O ConvLSTM pode ler as duas etapas de tempo e executar o processo da CNN nos sete dias de dados em cada uma.

Para esse problema, a entrada para o ConvLSTM2D seria, portanto:

[n, 2, 1, 7, 1]

Ou:

Samples: n, para o nmero de exemplos no conjunto de dados de treinamento. Time: 2, para as duas subsequncias em que dividimos uma janela de 14 dias. Rows: 1, para a forma unidimensional de cada subsequncia. Columns: 7, pelos sete dias em cada subsequncia. Channels: 1, para o recurso nico com o qual estamos trabalhando como entrada.

Voc pode explorar outras configuraes, como fornecer 21 dias de entrada divididos em trs subsequncias de sete dias e/ou fornecer todos os oito recursos ou canais como entrada.

    data = train.reshape((train.shape[0]*train.shape[1], train.shape[2]))
    X, y = list(), list()
    in_start = 0
    # percorrer toda a histria um passo de cada vez
    for _ in range(len(data)):
        # definir o final da sequncia de entrada
        in_end = in_start + n_input
        out_end = in_end + n_out
        # verifique se temos dados suficientes para esta instncia
        if out_end <= len(data):
            x_input = data[in_start:in_end, 0]
            x_input = x_input.reshape((len(x_input), 1))
            X.append(x_input)
            y.append(data[in_end:out_end, 0])
        # avanar um passo
        in_start += 1
    return array(X), array(y)

# treinamento do modelo
def build_model(train, n_steps, n_length, n_input):
    # preparacao dos dados
    train_x, train_y = to_supervised(train, n_input)
    # definicao dos parametros
    verbose, epochs, batch_size = 0, 20, 16
    n_timesteps, n_features, n_outputs = train_x.shape[1], train_x.shape[2], train_y.shape[1]
    # reshape no formato [samples, time steps, rows, cols, channels]
    train_x = train_x.reshape((train_x.shape[0], n_steps, 1, n_length, n_features))
    # reshape dos dados [samples, timesteps, features]
    train_y = train_y.reshape((train_y.shape[0], train_y.shape[1], 1))
    # definicao do modelo
    model = Sequential()
    model.add(ConvLSTM2D(filters=64, kernel_size=(1,3), activation='relu', input_shape=(n_steps, 1, n_length, n_features)))
    model.add(Flatten())
    model.add(RepeatVector(n_outputs))
    model.add(LSTM(200, activation='relu', return_sequences=True))
    model.add(TimeDistributed(Dense(100, activation='relu')))
    model.add(TimeDistributed(Dense(1)))
    model.compile(loss='mse', optimizer='adam')
    # fit do modelo
    model.fit(train_x, train_y, epochs=epochs, batch_size=batch_size, verbose=verbose)
    return model

# fazendo as previsoes
def forecast(model, history, n_steps, n_length, n_input):
    # achatamento dos dados
    data = np.array(history)
    data = data.reshape((data.shape[0]*data.shape[1], data.shape[2]))
    # recuperar ltimas observaes para dados de entrada
    input_x = data[-n_input:, 0]
    # reshape dos dados [samples, time steps, rows, cols, channels]
    input_x = input_x.reshape((1, n_steps, 1, n_length, 1))
    # prever a prxima semana
    yhat = model.predict(input_x, verbose=0)
    # buscando apenas a previso do vetor
    yhat = yhat[0]
    return yhat
